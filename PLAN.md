**사이트명:** 로또리 (Rottery.kr)

**언어:** 한국어

### 2.2 데이터 수집 전략

**로또 6/45 (공식 JSON API 존재):**
```
GET https://www.dhlottery.co.kr/common.do?method=getLottoNumber&drwNo={회차번호}
```
- 응답: JSON (drwtNo1~6, bnusNo, drwNoDate, firstWinamnt, totSellamnt 등)
- 1회부터 최신 회차까지 전체 이력 수집 가능

**연금복권/파워볼/스피드키노:**
- dhlottery.co.kr 웹페이지 스크래핑 (서버 사이드)
- 정기 크론잡으로 데이터 수집 및 DB 저장

## 3. 기술 스택 및 플랫폼 선택

### 3.1 추천 기술 스택: **Next.js (App Router) + Vercel**

| 항목 | 선택 | 이유 |
|------|------|------|
| **프레임워크** | Next.js 15 (App Router) | SSG/SSR 혼합, SEO 최적화, 빠른 로딩 |
| **호스팅** | Vercel (무료 티어) | Next.js 최적화, 자동 배포, 무료 SSL, 글로벌 CDN |
| **스타일링** | Tailwind CSS 4 | 빠른 개발, 반응형 디자인, 작은 번들 사이즈 |
| **언어** | TypeScript | 타입 안전성, 유지보수성 |
| **DB** | Vercel Postgres (또는 Supabase) | 무료 티어 충분, 당첨번호 이력 저장 |
| **ORM** | Drizzle ORM | 가볍고 타입세이프 |
| **데이터 수집** | Vercel Cron Jobs + API Routes | 자동 데이터 수집 |
| **분석** | Google Analytics 4 + Vercel Analytics | 트래픽/사용자 분석 |
| **광고** | Google AdSense | 주 수익원 |
| **도메인** | lottery.kr | 한국 도메인으로 네이버 SEO 유리 |

### 3.2 왜 Next.js + Vercel 인가?

1. **SEO 최적화**: SSG(Static Site Generation)로 복권별 페이지를 사전 생성 → 구글/네이버 크롤링 최적
2. **Core Web Vitals**: Vercel CDN + 이미지 최적화 → AdSense 승인에 유리
3. **무료 운영**: Vercel 무료 티어로 월 수만 PV까지 무료 운영 가능
4. **빠른 개발**: App Router의 서버 컴포넌트로 API 호출을 서버에서 처리
5. **Cron Jobs**: Vercel Cron으로 매주 당첨번호 자동 수집

### 3.3 대안 비교

| 대안 | 장점 | 단점 | 결론 |
|------|------|------|------|
| WordPress | 빠른 설정, 플러그인 많음 | 느림, 커스텀 어려움, 보안 취약 | ❌ 부적합 |
| Gatsby | 빠른 정적 사이트 | 빌드 느림, 동적 기능 제한 | ❌ 부적합 |
| Astro | 매우 빠름 | 동적 기능 부족, 생태계 작음 | △ 가능하나 차선 |
| **Next.js + Vercel** | SSG+SSR, SEO, 무료 호스팅 | 학습 곡선 | ✅ **최적** |

## 5. 번호 추천 알고리즘

### 5.1 추천 방식 (6가지)

| 방식 | 설명 | 구현 |
|------|------|------|
| **랜덤 추천** | 순수 랜덤 | `crypto.getRandomValues()` |
| **통계 기반** | 과거 데이터 빈도 분석 기반 가중치 랜덤 | 빈도 가중치 적용 |
| **핫넘버** | 최근 10~20회 자주 나온 번호 위주 | 최근 출현 빈도 상위 |
| **콜드넘버** | 오래 미출현된 번호 위주 | 미출현 기간 가중치 |
| **균형 추천** | 홀짝/고저/구간 균형 맞춤 | 제약조건 기반 생성 |
| **AI 추천** | 전체 통계 종합 분석 | 복합 가중치 알고리즘 |

### 5.2 AI 추천 알고리즘 상세

```typescript
// 복합 가중치 계산
score(number) =
  w1 * frequencyScore(number)      // 전체 출현 빈도 (0.2)
  + w2 * recentScore(number)       // 최근 20회 출현 빈도 (0.25)
  + w3 * coldScore(number)         // 미출현 기간 보상 (0.15)
  + w4 * patternScore(number)      // 연속/인접 패턴 (0.15)
  + w5 * sectionBalance(number)    // 구간 균형 (0.15)
  + w6 * oddEvenBalance(number)    // 홀짝 균형 (0.10)
  + random * 0.3                   // 랜덤 요소 (중요: 결정론 방지)
```

### 5.3 면책 조항 (필수)
